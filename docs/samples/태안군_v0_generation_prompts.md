# AI 화면 생성(v0) 프롬프트 가이드 (Prompt for v0.dev)

본 문서는 기획된 [태안군 모험도시 플랫폼]의 3개 주요 화면을 **v0.dev 등 AI UI 생성 도구**를 통해 퍼블리싱(Next.js/React & Tailwind CSS 등)하기 위해 최적화된 프롬프트 모음입니다. 

---

## 🏗️ [공통 룰] Base System Prompt (모든 화면 생성 시 필수 복붙)
v0 프롬프트 입력 시, 가장 상단에 본 시스템 프롬프트를 붙여넣어 전반적인 **UX/UI 톤앤매너 및 PRD 정책**을 세팅하세요.

```text
[역할] 
당신은 공공 스마트시티 통합 관제 플랫폼(Web)을 설계하는 최고 수준의 프론트엔드/UX 엔지니어입니다.

[핵심 UX 컨셉 & 테마: Spatial & Mobility Sync]
1. Theme: 반드시 'Dark Mode'를 기본 테마로 설정하세요. 배경은 깊은 네이비/차콜(#0f172a 또는 #1e293b) 계열을 사용해 장시간 관제 시 눈의 피로를 낮추세요.
2. Color System: 
   - Primary: Ocean Blue (활성화, 정상 데이터)
   - Secondary: Emerald Green (스테이션 등 고정된 '공간' 마커 색상)
   - Accent/Indicator: Electric Blue (버스/드론 등 이동하는 '모빌리티' 마커 색상)
   - Alert: Danger Red (장애, 에러, 통신 단절 시에만 제한적으로 사용) / Warning Yellow (지연 시 사용)
3. Layout Rules: Single Page Application(SPA)의 느낌을 주도록, 꽉 찬 레이아웃(Full-width)을 사용하고, 데이터 밀도가 높아도 Grid 시스템을 이용해 깔끔하게 모듈화하세요.
4. Exception Handling UI: 로딩 시에는 스켈레톤(Skeleton)이나 스피너(Spinner)를 주고, 에러/알림 발생 시 화면 최상단(Z-index 최고점)에 Toast 팝업 컴포넌트가 존재한다고 가정하고 여백을 두세요.
```

## 🖥️ 프롬프트 1. [화면 1] 메인 통합 관제 대시보드
**목적:** GIS 맵을 중앙에 둔 실시간 공간/이동 데이터 모니터링 화면 생성

```text
[요구사항]
위의 공통 룰을 바탕으로 "메인 통합 관제 대시보드" 화면을 만들어주세요.

[화면 레이아웃 구조]
- Header: 좌측에 '모험도시' 로고, 중앙에 네비게이션(통합 대시보드, 시스템 현황, 데이터 관리, 설정), 우측에 알림 종 모양 아이콘과 관리자 프로필.
- Center Area (Full View): 화면 전체를 덮는 거대한 어두운 테마의 GIS 지도(Placeholder 이미지나 간략한 SVG 지도 형태)를 배치하세요.
- Map Elements: 지도 위에는 초록색 점(스테이션 노드)과 파란색 선(모빌리티 궤적) 마커들이 놓여 있고, 하나의 마커는 빨간색 알림 테두리(Pulse 애니메이션) 효과를 띄고 있게 해주세요.
- Floating Controller (좌측 상단): 지도의 레이어를 끄고 켤 수 있는 토글 스위치(전체보기, 투어/케어 시설, 버스/드론 모빌리티, 장애/위험만 보기)를 살짝 투명한 글래스모피즘(Glassmorphism) 패널로 올려주세요.
- Right Widget Board (우측 사이드바): 차트 위젯들이 상하로 스택(Stack)된 패널을 배치하세요. 상단에는 톱니바퀴(편집) 아이콘이 있고, 아래로 '모빌리티 가동률(도넛 차트)', '시설물 통계(막대 차트)', '실시간 에러 피드(텍스트 리스트)' 위젯 컴포넌트들을 구성하세요.
```

## 🖥️ 프롬프트 2. [화면 2] 통합 시스템 및 서비스 현황 상황판
**목적:** 지도 없이 7종 서비스의 가동률과 IT 인프라 헬스를 보여주는 대시보드

```text
[요구사항]
위의 공통 룰을 바탕으로 "시스템 및 서비스 통합 현황판" 화면을 만들어주세요.

[화면 레이아웃 구조]
- Header: (화면 1과 동일한 헤더 유지)
- Top Section (Summary Cards): 화면 상단에 7개의 요약 통계 카드(Summary Cards)를 가로로 스크롤 가능하게 배치하세요. 카드에는 각각 '모험 드론', '모험 버스', '케어 스테이션' 등의 타이틀이 있고, 내부에 가동/정상 숫자는 초록색 뱃지로, 장애/에러 숫자는 빨간색 뱃지로 눈에 띄게 표시하세요. 카드는 호버(Hover) 시 살짝 떠오르는 느낌(Shadow 짙어짐)을 주세요.
- Bottom Left Section (IT Infrastructure): 하단 좌측 영역에는 플랫폼의 시스템 헬스 체크 위젯을 만드세요. 'WEB 서버 CPU/Memory 사용량', 'DB 서버 부하율'을 프로그레스(Progress) 바 형태로 보여주세요. 90%가 넘어가는 바는 빨간색으로 칠해주세요.
- Bottom Right Section (Event Log): 하단 우측 영역에는 '금일 주요 이벤트 및 장애 이력'을 나타내는 타임라인(Timeline) 컴포넌트 형태의 리스트를 만드세요. 제일 상단 행에는 오늘 13:10분에 발생한 붉은색 에러 텍스트 로그가 포함되도록 하세요.
```

## 🖥️ 프롬프트 3. [화면 3] 데이터 파이프라인(API) 관리 화면
**목적:** 수집되는 데이터 목록을 조회하고 장애 건을 제어하는 Admin 성격의 뷰

```text
[요구사항]
위의 공통 룰을 바탕으로 "데이터 파이프라인 관리" 화면을 만들어주세요.

[화면 레이아웃 구조]
- Page Title: "데이터 파이프라인 및 수집 관리" 
- Top Toolbar: 좌측에는 서비스 도메인, 상태(정상/오류), 수집일자를 필터링할 수 있는 콤보박스(Select) 폼을 배치하고, 우측에는 돋보기 아이콘이 있는 [조회] 버튼을 배치하세요.
- Main Data Grid (Table): 화면 중앙에 대용량 데이터 테이블을 만드세요. 컬럼은 [체크박스, 연계 서비스명, API/데이터명, 분류, 수집시간, 상태(🔴장애, 🟡지연, 🟢정상 뱃지), 제어 액션] 으로 구성하세요.
- Action Trigger (상호작용): 테이블에서 상태가 🔴장애로 표시된 Row의 끝에는 `[재수집 ↻]` 버튼이 있습니다. 버튼 중 하나는 클릭된 상태(비활성화+스피너 로딩 아이콘 렌더링)로 디자인해서 시스템이 비동기 동작 중임을 보여주세요.
- Drawer Panel (Exception Log): 화면 가장 하단 혹은 우측에서 살짝 밀려나온 서랍(Drawer) UI 패널을 만들어주세요. 패널 제목은 '[에러 상세 로그]' 이고 내부에는 JSON 형태의 응답 코드 스니펫(Syntax Highlight 된 코드 블록)이 들어있게 해주세요.
```
