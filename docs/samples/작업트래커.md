## 지금 순서대로 처리할 것 (Phase 5 착수 전 필수 2개)

### 1. ⬜ 팔레트 산출물 검증 로직 추가 — **담당: Claude Code**
**문제:** 시드 색 필터링(`isUsableBrandColor`)은 있지만, 그걸 통과한 뒤 실제로 생성되는 3세트 팔레트 자체가 여전히 짙은 남색·검정·회색 위주로 나옴 (2026-07-09 스크린샷 확인).
**지시 방향:** `lib/reference/palette.ts`의 팔레트 생성 함수에 산출물 검증 단계 추가 — 대비비(contrast ratio) 계산 + primary 계열 채도 하한선 체크. 기준 미달 시 재생성 또는 보정.
**완료 기준:** 3세트 중 최소 브랜드 충실형 세트에서 인디고/블루 계열처럼 채도 있는 색이 눈에 띄게 포함되는지 재확인.

### 2. ⬜ "펼쳐보기" 빈 내용 버그 — **담당: Claude Code**
**문제:** 분석 결과 화면, 구성 페이지 아코디언에서 "펼쳐보기" 클릭 시 내용이 안 나옴.
**지시 방향:** 렌더링 실패인지 데이터 자체가 비어있는지 원인 확인 → 다음 단계 진행(섹션 확정 게이트)에 영향 주는 버그인지 여부까지 같이 보고.
**완료 기준:** 원인 파악 + (다음 단계를 막는 버그라면) 수정까지.

### 4-1. ⬜ 페이지별 기능 테스트 — **담당: 부니** ← 지금 여기

**정정:** 이전에 "1~4단계 테스트 완료"로 기록했던 게 부정확했음. 실제로는 화면 캡처하다 눈에 띈 문제를 우연히 발견한 수준이었고, 시나리오 기반으로 체계적으로 확인한 건 아님.

| 단계 | 실제 상태 |
|---|---|
| 1. 업로드 | ✅ 테스트 완료 |
| 2. 마스킹 검수 | ✅ 테스트 완료 (버그 발견·수정) |
| 3. 분석 결과 | ⚠️ 깊은 테스트 안 됨 — 브랜드컬러 오인식·위계·펼쳐보기 빈내용만 우연히 발견 |
| 4. 레퍼런스·무드 | ⚠️ 깊은 테스트 안 됨 — 팔레트·인포박스 스타일 불일치만 우연히 발견 |
| 5. 컨셉 3안 | ⬜ 착수, 깊은 테스트 전 |
| 6. 디자인 MD | Phase 5 없어서 테스트 불가 |

**진짜 완료는 1~2단계뿐. 3~5단계는 전부 본격 테스트 필요한 상태.**
**확인 기준(3~5단계 공통):** 각 단계의 핵심 기능이 끝까지 동작하는지(다음 단계로 정상 진행되는지), 특히 5단계는 컨셉 확정 시 ConceptJson이 정상 생성되는지가 핵심. 화면이 안 예쁜 건 안정화 라운드(7~10번)로 미뤄도 됨.
발견되는 버그는 스크린샷으로 공유 → 같이 원인(데이터 결함 vs 화면 결함) 분류.
