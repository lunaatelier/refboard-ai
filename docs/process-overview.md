# 프로세스 개요 (스냅샷: 2026-07-04)

> 이 문서는 특정 시점의 진행 상황 스냅샷이다. 시간이 지나면 내용이 낡을 수 있으니,
> 실제 구현 순서·단위의 단일 기준은 항상 `docs/specs/implementation-steps.md`를 본다.
> 이 문서는 "지금 뭐가 됐고, 뭐가 남았고, 다음에 뭘 손볼지" 한눈에 보기 위한 참고용이다.

---

## 전체 데이터 흐름

```
[1] 업로드 (txt/md/pdf/pptx, 또는 저장한 분석 결과 JSON 재활용)
      ↓
[2] 마스킹 검수   ← Phase 1 (보안 하드 게이트 — 여기를 안 거치면 다음 단계 잠김)
      ↓
[3] 분석 결과    ← Phase 2 (도메인·페이지·섹션 확정)
      ↓
[4] 레퍼런스·무드 ← Phase 3 (팔레트·무드·섹션별 레퍼런스·분석 대상 브랜드)
      ↓
[5] 컨셉 3안     ← Phase 4 (Concept JSON = 단일 원천 → HTML/PPT/PDF)
      ↓
[6] 디자인 MD    ← Phase 5 (미착수 — 외부 표준 스키마 확정 대기)
```

## Step 1~13 — 전부 완료 (제품 A v1.0)

커밋 로그 기준 Step 1~13 모두 구현 완료. 업로드 → 마스킹 → 분석 → 레퍼런스 → 컨셉 3안 → 재활용모드까지 핵심 파이프라인이 전부 동작한다. 세부 스텝 내용은 `docs/specs/implementation-steps.md` 참조.
